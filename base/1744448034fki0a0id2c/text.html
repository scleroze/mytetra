План установки MyTetra и подключения к GitHub

1. Установка нужных программ

1.1. Установить git, ssh и саму mytetra:

sudo apt update

sudo apt install git openssh-client mytetra


(Если mytetra нет в репозиториях, скачать AppImage или tar.gz с сайта проекта.)

2. Настройка SSH для доступа к GitHub

2.1. Проверить наличие ключей:


ls ~/.ssh


2.2. Если ключей нет, создать новый:

ssh-keygen -t ed25519 -C "your_email@example.com"


На вопрос "куда сохранять" — просто нажать Enter (оставить стандартный путь).

2.3. Скопировать публичный ключ:


cat ~/.ssh/id_ed25519.pub


И добавить его на GitHub:

  • Открыть GitHub → Settings → SSH and GPG keys → New SSH key,
  • Вставить туда содержимое id_ed25519.pub.

2.4. Проверить подключение к GitHub:


ssh -T git@github.com


Ожидаемый ответ:

Hi your_username! You've successfully authenticated, but GitHub does not provide shell access.


3. Подключение репозитория MyTetra

3.1. Перейти в папку настроек MyTetra (создать её, если пустая):


mkdir -p ~/.config/mytetra

cd ~/.config/mytetra


3.2. Клонировать свой репозиторий с GitHub:


git clone git@github.com:scleroze/mytetra.git ~/.config/mytetra/data


(замени your_username и your_repo на свои).

В результате должна появиться папка ~/.config/mytetra/data с данными.

4. Настройка сценария синхронизации в MyTetra

4.1. Открыть настройки MyTetra.

4.2. В разделе "Сценарий синхронизации данных" прописать:

bash

КопироватьРедактировать

cd %a ; git add . ; git commit -a -m MyTetraCommit ; git pull --rebase ; git push


4.3. Сохранить настройки.

🎯 Всё!

Теперь при нажатии кнопки "Синхронизировать" в MyTetra:

  • все локальные изменения автоматически добавляются, коммитятся и отправляются на GitHub.
